# 프로필 상세 화면 설계

**날짜:** 2026-06-09
**상태:** 확정

## 개요

로그인한 사용자가 자신의 프로필 정보를 조회하는 화면. 사용자 관리(`/account/user/[id]`)와 완전 분리된 독립 페이지로 구성.

## 라우트 & 접근 경로

- **URL:** `/profile`
- **접근:** 헤더의 유저 아이콘/이름 클릭
- **수정 버튼:** 클릭 시 `/profile/edit` 이동 (수정 화면은 별도 개발)

## 분리 원칙

| 구분 | 프로필 상세 `/profile` | 사용자 상세 `/account/user/[id]` |
|------|----------------------|--------------------------------|
| 주체 | 본인 | 관리자가 타인 관리 |
| 데이터 소스 | Jotai `auth.store` atoms | API 단건 조회 |
| 편집 범위 | 본인 기본 정보 | grade, status 등 관리 항목 포함 |

## 표시 필드

| 필드 | 레이블 | 위치 | 비고 |
|------|--------|------|------|
| `avatar` | — | 상단 좌측 이미지 | |
| `name` | 이름 | 아바타 우측 | |
| `email` | — | 이름 하단 | |
| `grade` | 등급 | 이메일 하단 | 뱃지 형태 표시 |
| `phone` | 연락처 | 하단 필드 목록 | |
| `company` | 소속 | 하단 필드 목록 | 외부/내부 구성원 모두 포함하는 명칭 |
| `bio` | 소개 | 하단 필드 목록 | |

**제외 필드:** `location` (위치)

## 데이터 흐름

- 로그인 시 `setUserInfoAtom`으로 세팅된 Jotai atoms를 직접 읽어 표시
- 별도 API 호출 없음
- 사용 atoms: `nameAtom`, `emailAtom`, `avatarAtom`, `phoneAtom`, `companyAtom`, `bioAtom`, `gradeAtom` (`src/features/auth/store/auth.store.ts`)
